## দিন ১ — সাইট লাইভ + Admin চেনা

| করবেন | করবেন না |
|--------|-----------|
| `admin-login.html` খুলে লগইন টেস্ট | D ড্রাইভ থেকে `file://` দিয়ে সাইট চালানো |
| **admin-links.html** — সব লিংক একবার ক্লিক করে দেখা | cPanel খোলা (Netlify ব্যবহার করলে) |
| API URL খুলে `Muslim Abaya API OK` দেখা | পাসওয়ার্ড chat/WhatsApp-এ পাঠানো |
| Sheet-এ ২–৩টা পুরনো অর্ডার row চেনা | WordPress ইনস্টল ভাবা |

**সময়:** ~২০ মিনিট

---

## দিন ২ — অর্ডার রুটিন (মূল কাজ)

| সকাল/দিনে | |
|------------|--|
| Sheet → **Online Order** → নতুন row | |
| **Pending** অর্ডার → **ফোনে confirm** (নাম, ঠিকানা, টাকা) | |
| Confirm → Status **Confirmed** (কলাম J) | |
| সন্দেহ হলে অর্ডার বাতিল / আরেকবার কল | |

| করবেন না |
|----------|
| প্রতিটি অর্ডার শুধু Sheet ছাড়া কাগজে রাখা |
| confirm ছাড়াই Steadfast পাঠানো |

**সময়:** প্রতিদিন ১৫–৩০ মিনিট (অর্ডার অনুযায়ী)

---

## দিন ৩ — Admin ড্যাশবোর্ড + টেস্ট অর্ডার

| করবেন | |
|--------|--|
| `admin-login` → **admin-dashboard** — অর্ডার তালিকা দেখা | |
| সাইট থেকে **১টা টেস্ট অর্ডার** (নিজের নম্বর, ছোট টাকা) | |
| ২–৩ মিনিট পর Sheet + ড্যাশবোর্ডে row আছে কিনা | |
| টেস্ট row-এ Notes-এ `TEST` লিখে রাখা | |

**সময়:** ~১৫ মিনিট

---

## দিন ৪ — Steadfast + ট্র্যাকিং (key থাকলে)

| করবেন | |
|--------|--|
| Script properties-এ Steadfast key আছে কিনা (Apps Script → Project Settings) | |
| Sheet → row সিলেক্ট → **Muslim Abaya → Steadfast পাঠান** | |
| Tracking (K) পেলে গ্রাহককে WhatsApp/কল | |
| **Steadfast — balance** মেনু একবার চেক | |

| key না থাকলে |
|---------------|
| Steadfast প্যানেল থেকে key নিন → Script properties-এ সেট করুন (chat-এ পাঠাবেন না) |

**সময়:** ~২০ মিনিট (প্রথমবার), পরে ৫ মিনিট/অর্ডার
